# TG WS Proxy

**TG WS Proxy** — локальный MTProto-прокси для Telegram Desktop, работающий поверх WebSocket. Такой механизм позволяет организовать стабильную передачу данных между клиентом и серверами Telegram без использования сторонних прокси, сохраняя стандартное MTProto-шифрование.

# Установка для Windows

Перейдите в раздел **Releases** и загрузите исполняемый файл, подходящий для вашей операционной системы.

Доступны следующие сборки:

- `TgWsProxy_windows.exe` — Windows 10 и новее (x64);
- `TgWsProxy_windows_arm64.exe` — Windows 10 и новее (ARM64);
- `TgWsProxy_windows_7_64bit.exe` — Windows 7 (x64);
- `TgWsProxy_windows_7_32bit.exe` — Windows 7 (x86).

При первом запуске программа автоматически откроет инструкцию по подключению Telegram Desktop. После завершения первоначальной настройки приложение продолжит работу в фоновом режиме, а управление будет доступно через значок в системном трее.

## Меню программы

Все основные функции доступны через контекстное меню значка TG WS Proxy.

Доступные действия:

- **Открыть в Telegram** — автоматически создать ссылку `tg://proxy` и передать её клиенту Telegram.
- **Скопировать ссылку** — сохранить ссылку подключения в буфер обмена.
- **Перезапустить прокси** — повторно запустить локальный MTProto-прокси без закрытия приложения.
- **Настройки...** — открыть окно конфигурации, изменить параметры работы и включить автоматическую проверку обновлений.
- **Открыть журнал** — просмотреть журнал событий приложения.
- **Выход** — полностью завершить работу TG WS Proxy.

# Подключение Telegram Desktop

## Автоматическая настройка

Для большинства пользователей рекомендуется использовать автоматический способ подключения.

Выполните следующие действия:

1. Щёлкните правой кнопкой мыши по значку TG WS Proxy в области уведомлений.
2. Выберите пункт **«Открыть в Telegram»**.
3. Подтвердите подключение прокси.

Если ссылка не открылась автоматически:

1. Откройте меню программы.
2. Выберите **«Скопировать ссылку»**.
3. Отправьте её в чат **«Избранное»**.
4. Откройте отправленную ссылку.
5. Подтвердите применение настроек.

## Ручная настройка

Если автоматический способ подключения недоступен, необходимые параметры можно указать самостоятельно.

Откройте раздел:

```text
Telegram
→ Настройки
→ Продвинутые настройки
→ Тип подключения
→ Прокси
```

Создайте новое MTProto-подключение и заполните поля:

| Параметр | Значение |
|----------|----------|
| Тип | MTProto |
| Сервер | `127.0.0.1` (или адрес, указанный в настройках приложения) |
| Порт | `1443` (либо пользовательское значение) |
| Secret | секретный ключ из конфигурации приложения или журнала логов |
# Принцип работы

```text
Telegram Desktop
        │
        ▼
Локальный MTProto Proxy
        │
        ▼
WebSocket (TLS)
        │
        ▼
Telegram Data Center
```

После запуска TG WS Proxy принимает локальные подключения от Telegram Desktop и самостоятельно организует передачу трафика к серверам Telegram через защищённый WebSocket-канал.

Последовательность работы включает следующие этапы:

1. Инициализация локального MTProto-прокси с параметрами, заданными в конфигурации.
2. Приём входящих соединений от Telegram Desktop.
3. Анализ **MTProto Obfuscation Init Packet** для определения идентификатора требуемого дата-центра Telegram (DC ID).
4. Создание защищённого WebSocket-соединения с соответствующим сервером Telegram.
5. При невозможности установить WebSocket-соединение (например, если сервер возвращает **HTTP 302 Redirect**) приложение автоматически переключается на резервный маршрут — Cloudflare Proxy либо прямое TCP-подключение.

> [!IMPORTANT]
>
> ### Не открываются изображения или видео?
>
> Если после настройки TG WS Proxy Telegram перестал загружать фотографии, видеоролики или другой медиаконтент, сначала проверьте конфигурацию IP-адресов дата-центров.
>
> Откройте раздел:
>
> **DC → IP**
>
> и оставьте только следующую запись:
>
> ```text
> 4:149.154.167.220
> ```
>
> Если после этого проблема сохраняется, полностью очистите содержимое поля и повторите попытку подключения.
>
> Подобная особенность иногда встречается на аккаунтах без активной подписки **Telegram Premium**.
>
> Если перечисленные действия не помогли, рекомендуется использовать собственный Cloudflare Proxy, настроив его по инструкции:
>
> **[CfProxy.md](./CfProxy.md)**

# Автоматическая сборка

В репозитории уже присутствуют все необходимые конфигурационные файлы для автоматической подготовки исполняемых сборок.

Для упаковки приложения применяется **PyInstaller**, а процесс создания релизов полностью автоматизирован с помощью **GitHub Actions**.

Используемые файлы конфигурации:

- `packaging/windows.spec`
- `packaging/macos.spec`
- `packaging/linux.spec`

Workflow автоматической сборки расположен по адресу:

- `.github/workflows/build.yml`

Такой подход позволяет автоматически формировать актуальные версии приложения после внесения изменений в исходный код.
